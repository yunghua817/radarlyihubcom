Claude + Claude Code

Claude 用法

不只聊天。Claude 可以拿來做研究、整理工作區、養大 Projects、做 Artifacts、 生出 Excel / PPT / PDF，也能用 Claude Code 讀倉庫、修 bug、拆 subagent 和掛 hooks。

Claude 用法案例池

從研究到工程，這 12 張卡都能直接複製提示詞起跑。

01研究模式

讓 Claude 做多來源研究備忘錄

能實現的效果

跨網頁與可用內部資料做研究，輸出帶引用的決策摘要。

複製給 Claude

請你使用 Research 幫我研究「[主題]」。先搜尋網頁與可用內部來源，整理 5 個候選方案的定位、功能、價格、風險、適用情境與仍待確認事項，最後輸出成一頁決策備忘錄，所有關鍵結論都要附來源。

官方功能 Research Web Search
02會議戰情

讓 Claude 幫你做會前 briefing

能實現的效果

從 Gmail、Calendar、Drive 拉出上下文，整理成會前摘要。

複製給 Claude

請結合我的 Gmail、Google Calendar 與 Google Drive，幫我為「[會議名稱]」做一份 10 分鐘會前 briefing：整理最近往來郵件、相關文件、未完成行動項、潛在風險與建議決策，最後用條列式輸出。

官方教學 Google Workspace Research + Workspace
03專案知識庫

把 Claude Project 養成你的長文件專案室

能實現的效果

把 PRD、會議記錄、規格與研究資料放進同一個知識空間。

複製給 Claude

我會把 PRD、會議記錄、訪談摘要、技術筆記與競品資料都放進這個 Claude Project。請先幫我建立一份「目前產品狀態」摘要：列出關鍵名詞、已決事項、待確認事項、資訊缺口，以及下一次討論前需要補齊的資料。

官方支援 Projects Projects RAG
04Artifacts 原型

讓 Claude 把需求變成可互動原型

能實現的效果

把 PRD、流程或算式，變成能點、能看、能分享的 demo。

複製給 Claude

請把這份需求整理成一個可互動的 Artifact 原型。先定義使用者流程與輸入輸出，再做一個能展示核心價值的單頁 demo；介面要讓非工程同事也能一眼理解，並附上後續可延伸成正式產品的建議。

官方功能 Artifacts File Creation
05檔案產出

讓 Claude 一次產出 Excel、PPT、Word、PDF

能實現的效果

把資料、說明與報告一次打成可下載的交付包。

複製給 Claude

請你根據我上傳的資料，幫我做一個交付包：1 份 Excel 分析表、1 份 PowerPoint 簡報、1 份 Word 摘要、1 份 PDF 匯出版本。請先列出每個檔案要包含的內容，再開始生成，並保持格式專業、中文可直接對外使用。

官方功能 Create Files Upload Files
06連接器助手

讓 Claude 自己判斷該叫哪些 connector

能實現的效果

跨 Slack、Linear、Drive、日曆等系統找資料，但先以只讀為主。

複製給 Claude

請先判斷這個任務最適合用哪些 connector 取資料，例如 Slack、Linear、Drive、Calendar 或其他已接入服務。優先用只讀方式幫我整理上下文；如果要回寫任何系統，只先產生草稿並說明風險，再等我確認。

官方功能 Connectors Google Workspace
07聊天記憶

讓 Claude 幫你接回上次聊到哪

能實現的效果

從過往聊天裡把決策、脈絡和未完成事項找回來。

複製給 Claude

請幫我搜尋我們之前聊過和「[主題]」有關的對話，把核心結論、尚未完成的事情、已做決策和你認為我下一步最適合做的事整理成一份續接摘要，讓我不用自己回頭翻聊天紀錄。

官方功能 Chat Search Projects & Caching
08倉庫導覽

讓 Claude Code 先幫你讀懂一個 repo

能實現的效果

快速認識技術棧、入口點、模組分工與關鍵風險。

複製給 Claude Code

請你作為倉庫導覽員，先不要改程式。先閱讀這個 repo 的目錄、README、套件設定、進入點和核心模組，告訴我：這個專案在做什麼、怎麼啟動、哪些模組最重要、最可能卡在哪裡，以及我第一次應該先看哪 5 個檔案。

Claude Code Docs Overview Quickstart
09修 bug / 做功能

讓 Claude Code 分段推進修復與實作

能實現的效果

先定位、再改動、再驗證，不是一口氣亂改整個專案。

複製給 Claude Code

請你作為 Claude Code 工程搭檔，先分析「[bug 或功能需求]」的上下文，寫出診斷假設與驗證步驟，再用最小改動完成修復或功能實作；完成後請列出你改了哪些檔案、怎麼驗證、還有哪些殘留風險。

Claude Code Docs Common Workflows Quickstart
10專案規則

讓 Claude Code 幫你整理 CLAUDE.md

能實現的效果

把專案規則、命令、禁忌與驗收方式沉澱成長期記憶。

複製給 Claude Code

請你幫我為這個專案整理一份 CLAUDE.md。先閱讀 README、常用命令、測試方式和工作流，再寫出：專案目標、開發指令、注意事項、不可擅自執行的動作、驗收步驟，以及對未來自己最有幫助的背景知識。

Claude Code Docs CLAUDE.md & Memory Project Context
11Subagents / Skills

把大任務拆成專用小幫手

能實現的效果

研究、找 log、審查、產文檔分開跑，主線上下文更乾淨。

複製給 Claude Code

請判斷這個任務哪些部分應該交給 subagent，哪些應該包成 skill。先提出分工方案，例如：架構探索、log 蒐集、程式碼審查、文件整理各自怎麼拆，並說明這樣拆可以幫我避免哪些上下文污染與重複工作。

Claude Code Docs Subagents Skills
12Hooks 自動化

讓 Claude Code 自動格式化、提醒、保護檔案

能實現的效果

把固定規則做成 hooks，讓流程更穩，不用每次口頭提醒。

複製給 Claude Code

請你為這個專案設計一套 Claude Code hooks：檔案改完自動格式化、等待我回覆時通知我、保護敏感檔案不被誤改、必要時重新注入上下文。先用表格說明每個 hook 何時觸發、做什麼、風險是什麼，再給我設定草案。

Claude Code Docs Hooks Guide Hooks Reference
